tsort :: Ord k => (v -> [k]) -> Tree k v -> [[k]]

Topological sort of a Tree

The set of objects is the set of values in the Tree, identified by their keys.

There must be a function f that takes a value and computes a list of keys that denote the values direct dependencies. Dependencies that are not elements of the tree are ignored.

The result is a list of list of keys, where the keys of mutually dependent values are grouped in one sublist and, for all values, the keys of all transitive dependencies of that value are in sublists that appear before the sublist that contains the key of that value.

Hence, if the value indicated by key a depends on the value indicated by key b, b will not appear later than a in the output. And, if a and b are in the same sublist, their values are mutually dependent.

If the output has only singleton lists, there are no mutual dependencies.

 tree = Tree.fromList [('a', 1), ('b', 2), ('c', 3), ('d',4)]
 f 1 = ['d', 'b']
 f 2 = ['c']
 f 3 = ['d']
 f 4 = ['c']
 f _ = []
 tsort tree = [['c', 'd'], ['b'], ['a']]
